60 lectures, a note for everyone!

1. Motivation for Graph ML

  • capture the relationships between different objects or entities
  • graph structure allows us to model the underlying domain in a fateful way
  • representation learning: automatically learn the features
  • learn a neurak network: f u–> R(d)(d dimension)

2. Applications of Graph ML

  • node classification, link prediction, graph classification, Clustering(detect if nodes form a community)
  • Graph generation: drug discovery; Graph evolution: Physical simulation

3. Choice of Graph Representation

  • Components: Objects, interactions, systems
  • Bipartite graph: Authors to papers, Actors to movies(appeared in), users to movies(rated on)
    • folded/projected bipartite graph: link the nodes if they have the same neighbor(thus U, V get connected, which is called a project)
  • Representation of graph: Adjacency matrix, Edge list(2-d array)
    • most real-world networks are sparse
  • Node and Edge attributes

4. Traditional Feature-based Methods: Node-level features

  • traditional ML pipelines uses hand-designed features
  • two types of features: structural features, attributes of the node itself
  • Goal: charaterize the position and the structure of a node in the graph
    • method: Node degree, Node centrality, Clustering coefficient, Graphlets
  • Node degree drawback: counts the neighboring nodes without capturing the importance of them
  • Node centrality tries to capture the importance
    • Eigenvector centrality: λc = Ac, the leading eigenvector c is for centrality
    • Betweeness centrality: more important if lies on more shortest paths
    • Closeness centrality: A node is important if it has small shortest path length to all other nodes
    • Clustering coefficient: how connected the neighbors are
  • We can see that Clustering coefficient counts the triangles, generalized–> graphlets
  • Graphlet Degree Vector(GDV): a count vector of graphlets rooted at a given node
    • provide a measure of a node’s local network topology
    • more detailed, and many hops
  • two types of features:
    • importance-based features(predict important nodes,e.g. celebrity in social network)
      • Node degree
      • Node centrality
    • structure-based features(predict the role of a node in the graph)
      • Node degree
      • Clustering coefficient
      • Graphlet degree vector

5. Traditional Feature-based Methods: Link-level features

  • predict new links based on existing links, top k node pairs are predicted while testing
  • the key is to design features for a pair of nodes
  • two formulaions of the task
    • Links missing at random
    • Links over time
  • features:
    • distance-based features
      • shortest path
      • local neighborhood features(shared neighbors…), but value for nodes don’t have common neighbors is 0, which may also be connected in the future
      • global neighborhood overlap: Katz index(count the num of all lengths of paths between a pair of nodes, using powers of A)

6. Traditional Feature-based Methods: Graph-level features

  • Kernel methods are widely used in traditional ML
  • Graph Kernels: measure similarity between 2 graphs
  • Bag-of-*
  • Color Refinement: after k steps of color refinements, c(k+1)(v) summarizes the the structure of k-hop neighborhood, hash the color, then the WL kernel counts the number of nodes of a given color, then compute the inner product of the two color count vectors, time complexity linear in #edges

7. Node Embeddings

  • Representation learning: automatically
  • Task: map nodes into an embedding space
  • Encoder: maps each node to a low-dimensional vector
  • Decoder: similarity function: dot product and so on
  • Simplest encoding approach:Encoder is just an embedding-lookup(shallow encoder, optimize the embeddings directly)
  • Node embeddings: self-supervised, task independent

8. Random Walk Approaches for Node Embeddings

  • expressive(idea: if r.w. from u visits v with high probability, u v are similar(high-order multi-hop information)), efficient(not all the pairs of node)
  • instead of normalizing w.r.t.(with respect to) all nodes, just normalize against k random “negative samples”
  • higher k gives more robust estimates, in pratice k = 5~20
  • stochastic gradient descent, zi are all initailized with random values
  • Summary: run short fixed length randon-walk starting from each node, for each node u collect N(u), write the loss function, optimize the embeddings with SGD. Loss function can be efficiently approximated with negative sampling.
  • random walk: simplest idea:just run fixed length, unbiased(DeepWalk KDD’14)
  • node2vec(KDD’16)
  • goal: Embed nodes with similar network neighborhoods close in the feature space(frame this goal as a maximum likelihood optimization problem)
  • Idea: use flexible, biased random walk, trade off between local and global(bfs | dfs)
  • Two parameters
    • Retuen parameter p(to the previous node)
    • In-out parameter q:(moving outwards(DFS) or inwards(BFS))
    • 1/p back, 1/q out, 1 walk to a node with the same distance to the previous node (1/p, 1/q, 1) are unnormalized
    • come over edge(s1,w) and is at w, corresponding dist is(0, 2, 1)
    • BFS-like walk: Low value if p
  • Linear-time complexity, 3 steps are parallelized

9. Embedding Entire Graphs

  • Tasks: Classifying toxic vs. non-toxic molecules
  • Idea1: sum(or average) the embeddings of all the nodes
  • Idea2: Introduce a “virual node”
  • Anonymous Walk(ICML’18)
  • States in anonymous walks correspond to the index of the firt time we visited the node in the random walk
  • Represent the graph as a probability distribution over anonymous walks
  • For example, set l = 3, then we can represent the graph as a 5-dim vector, since there are 5 anonymous walks of length 3(111,112,121,122,123)
  • New idea: Learn Walk Embeddings(a little confused)

10. PageRank

  • A page is important if it is pointed by other important pages
  • rj = sum(ri / di)
  • M is the stochastic adjacency matrix M, wanna solve r = Mr

11. How to Solve?

  • Power Iteration Method
  • Random telepot: jump to any pages with prob. β, solves the issues of dead-ends(some pages have no out-links, cause importance to “leak out”) and spider-traps(all out-links are within the group, eventually sppider traps absorb all importance)

12. Random Walk with Restarts and Personalized Pagerank

  • Ranks proximity of nodes to the teleport nodes S

  • Q: What’s most related to Item Q? Let S = {Q}

  • In traditional Pagrank, S is all of the nodes

  • Idea:

    • Every node has some importance
    • Importance gets evenly split among all edges and pushed to the neighbors

  • Random Walk with Restarts: Personalized Pagerank where teleport is the same node

  • Count the number of visits, if is bigger, the nodes are more similar to start node S, as their are more common neighbors and the distance is short(with less restart)

  • Benefits: considers:

    • Multiple connections
    • Multiple paths
    • Direct and inderect connections
    • Degree of the node
